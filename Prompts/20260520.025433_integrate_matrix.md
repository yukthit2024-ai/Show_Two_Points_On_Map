# Implementation Plan - Matrix Chat Protocol Integration & Left Drawer Navigation

This plan details the architecture and step-by-step implementation for adding Matrix chat room polling and drawer-based settings navigation to the Friend Location Tracker application.

## 1. Objectives & UI/UX Vision
- **Sleek Navigation Drawer**: Use a modern dark drawer overlay (`DrawerLayout`) with premium custom icons and typography.
- **Glassmorphic Menu Pages**: Implement elegant, slide-up dialog bottom sheets or screen overlays for Settings, Matrix Credentials, Matrix Rooms, Help, and About.
- **Shared Preferences Persistence**: Save settings and credentials securely. Store Matrix rooms as a JSON array of objects.
- **Matrix Network Client**: Implement a custom asynchronous `MatrixClient` using `OkHttpClient` and built-in Android JSON library to:
  - Automate username/password authentication (obtaining and saving an access token).
  - Periodically poll registered rooms using `GET /_matrix/client/v3/rooms/{roomId}/messages?dir=b&limit=5`.
  - Filter new messages using the highest processed `origin_server_ts` timestamp.
  - Dispatch messages to a callback to "do some stuff" (e.g. show a premium custom Snackbar or Toast).

## 2. Technical Component Design

### A. Layout Enhancements (`activity_main.xml`)
We will transform the root `FrameLayout` into a `androidx.drawerlayout.widget.DrawerLayout`.
- **Main Panel**:
  - `MapView` filling the screen.
  - Floating top-left Hamburger button (using a beautiful custom vector or drawable icon) to open the drawer.
- **Drawer Panel**:
  - Left gravity (`android:layout_gravity="start"`).
  - Premium dark layout with app branding, list items with visual highlights, and icons.

### B. Settings Storage
Create a `MatrixPrefs` helper or utilize `SharedPreferences` in `MainActivity.java`:
- `polling_interval` (default: 1 second)
- `matrix_homeserver` (default: `https://matrix.org`)
- `matrix_token` (persistent access token)
- `matrix_username`
- `matrix_password`
- `matrix_display_name`
- `matrix_polling_period` (default: 5 seconds)
- `matrix_rooms` (JSON Array containing `{ "name": "...", "id": "..." }`)

### C. Dialog Overlays
Implement beautiful Material dialog sheets for:
1. **Settings**: Text field for "Polling Interval".
2. **Matrix Credentials**: Form containing text inputs for homeserver, token, username, password, display name, and polling period. Includes a "Test Login" / "Save" action.
3. **Matrix Rooms**: List view showing columns (Room display name | Room identifier) and an "Add Room" custom dialog form, along with delete operations.
4. **Help**: Scrollable explanation page.
5. **About**: Premium details displaying application metadata.

### D. Matrix Client & Background Polling
- Create an asynchronous polling runnable driven by the "Polling period" setting.
- Keep track of the last processed event timestamps per room to prevent duplicate event dispatches.

- [ ] Create beautiful vector icon drawables for Hamburger menu and list items.
- [ ] Implement settings screens using Material dialog builders.
- [ ] Develop the custom lightweight HTTP client for the Matrix protocol.
- [ ] Run the background loop for polling and handle received messages.
- [ ] Compile and verify building successfully.
