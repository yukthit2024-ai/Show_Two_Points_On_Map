# Implementation Plan - Fixing Marker Visibility and Snapping Behavior

This plan outlines the steps to resolve the marker visibility issues and eliminate the intrusive camera snapping behavior in the 1-second loop.

## 1. Marker Visibility (Tinted Project Assets)
Instead of generating custom bitmaps programmatically from scratch, we will load the project's existing high-quality marker assets (`R.drawable.ic_friend_marker`), tint them programmatically using Android's `DrawableCompat` to Red and Green, and add them to the map style **before** initializing the `SymbolManager`. This ensures perfect rendering compatibility, asset scaling, and absolute visibility.

## 2. Eliminate Camera Snapping
In the repeating `movementRunnable` loop, we will remove the `mapLibreMap.animateCamera(CameraUpdateFactory.newLatLng(midpoint))` call. 
- *Why*: Centering the camera on the midpoint every 1 second makes it impossible for the user to pan or move around the map, as any drag is instantly snapped back to the Cochin midpoint.
- *Solution*: We will only position the camera initially upon startup using `moveCamera(newLatLngZoom(midpoint, 14.5))`. In the loop, we will update the marker positions smoothly on the map, allowing the user to pan, zoom, and explore the map freely without any snapping.

## Technical Details

### 1. Tinted Bitmap Helper Method
We will add `getTintedMarkerBitmap` to `MainActivity.java`:
```java
private Bitmap getTintedMarkerBitmap(int drawableId, int tintColor) {
    Drawable drawable = androidx.core.content.ContextCompat.getDrawable(this, drawableId);
    if (drawable == null) {
        return createMarkerBitmap(tintColor); // Fallback to programmatic circle
    }
    
    // Wrap and mutate to apply tint color safely
    drawable = androidx.core.graphics.drawable.DrawableCompat.wrap(drawable).mutate();
    androidx.core.graphics.drawable.DrawableCompat.setTint(drawable, tintColor);
    
    int width = drawable.getIntrinsicWidth() > 0 ? drawable.getIntrinsicWidth() : 128;
    int height = drawable.getIntrinsicHeight() > 0 ? drawable.getIntrinsicHeight() : 128;
    
    // Maintain maximum bounds for rendering optimization
    int maxSize = 128;
    if (width > maxSize || height > maxSize) {
        float ratio = Math.min((float) maxSize / width, (float) maxSize / height);
        width = Math.round(width * ratio);
        height = Math.round(height * ratio);
    }
    
    Bitmap bitmap = Bitmap.createBitmap(width, height, Bitmap.Config.ARGB_8888);
    Canvas canvas = new Canvas(bitmap);
    drawable.setBounds(0, 0, canvas.getWidth(), canvas.getHeight());
    drawable.draw(canvas);
    
    return bitmap;
}
```

### 2. Startup Setup Order
We will modify the style loaded callback to:
1. Generate Red and Green tinted marker images from `R.drawable.ic_friend_marker`.
2. Add them to style: `style.addImage("red-marker", ...)` and `style.addImage("green-marker", ...)`.
3. Initialize `SymbolManager` with the loaded style.
4. Position the camera once using `moveCamera`.
5. Start the movement loop.
