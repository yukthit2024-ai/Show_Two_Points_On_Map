# User Prompt
Instead of generating random GPS locations, read the GPS locations from text files under /sdcard/Vypeensoft/Friends_Location_Tracker/sessions/
Each user's location is specified in a separate file inside that folder
If there are 2 files in that folder, then two markers are to be shown on map.
If there are 5 files in that folder, then five markers are to be shown on the map.  The Color field is used to spcify which color marker to be displayed.

Each file has these details, separated by PIPE character
Username|Latitude|Longitude|Color

# Implementation Plan - Dynamic File-Based GPS Tracker

This plan details the steps to modify `MainActivity.java` and `AndroidManifest.xml` to support loading and dynamically updating map markers based on location files found in `/sdcard/Vypeensoft/Friends_Location_Tracker/sessions/`.

## 1. Objectives & Requirements
- **Dynamic File Reading**: Stop generating random movement/positions. Instead, scan the target folder `/sdcard/Vypeensoft/Friends_Location_Tracker/sessions/` for files.
- **Dynamic Marker List**: Support any number of markers (e.g. 2, 5, etc.) based on the number of files.
- **Pipe-Separated Content**: Read and parse `Username|Latitude|Longitude|Color` from each file.
- **Dynamic Colored Badges**: Reuse the existing `createTeardropMarkerBitmap(color, text)` function to render beautiful markers with the custom name and color.
- **Runtime Permissions**: Add checking and requesting for Storage/Manage Files permission to access direct `/sdcard` paths.
- **Periodic Update/Polling**: Run a polling routine (e.g. every 1-2 seconds) to sync the map markers with changes to the files.

## 2. Technical Solution Detail

### A. AndroidManifest.xml updates
Add storage access and all-files management permission:
```xml
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.MANAGE_EXTERNAL_STORAGE"
    tools:ignore="ProtectedPermissions" />
```

### B. MainActivity.java updates
1. **Dynamic Marker State Tracking**:
   Use a `Map<String, Marker>` to store and manage active markers dynamically.
2. **Permission Handling**:
   Check if the app has required storage access permissions. If missing on Android 11+ (API 30+), trigger standard System Activity for All Files Access (`Settings.ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION`).
3. **Polling/Update Mechanism**:
   Modify the repeating Handler loop (`startMovementLoop`) to:
   - Request permissions if not granted.
   - Scan target directory `/sdcard/Vypeensoft/Friends_Location_Tracker/sessions/`.
   - Create the directory and write sample files if empty so the app has working demonstration markers.
   - Parse each file.
   - Diff the existing `Map<String, Marker>` with the parsed files: remove deleted users, add new users, and update positions/colors of existing ones.
   - Center/Zoom the camera around the active markers on the first successful load.
