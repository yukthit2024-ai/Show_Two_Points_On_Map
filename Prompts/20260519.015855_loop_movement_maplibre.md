# Implementation Plan - Dynamic 1-Second Loop Movement

This plan details the steps to implement a repeating 1-second animation loop in `MainActivity.java` that shifts both the Red and Green markers by exactly 1 meter in a random direction at each tick, while preserving their exact 1-kilometer relative separation.

## Proposed Strategy

1. **State Preservation**:
   Store the active coordinates and symbol references as instance fields in `MainActivity`:
   - `private LatLng redLatLng;`
   - `private LatLng greenLatLng;`
   - `private org.maplibre.android.plugins.annotation.Symbol redSymbol;`
   - `private org.maplibre.android.plugins.annotation.Symbol greenSymbol;`

2. **Loop Handler Setup**:
   Define a `Handler` and a `Runnable` update loop:
   - `private final android.os.Handler movementHandler = new android.os.Handler(android.os.Looper.getMainLooper());`
   - `private Runnable movementRunnable;`
   - Loop interval: `1000L` (1 second).

3. **Mathematical Vector Translations**:
   At each tick (every 1 second):
   - Choose a random bearing $\alpha \in [0, 2\pi]$.
   - Compute the delta offset for **1 meter** ($0.001\text{ km}$):
     - $\Delta\text{Lat}_{1m} = (0.001 / 111.12) \times \cos(\alpha)$
     - $\Delta\text{Lon}_{1m} = ((0.001 / 111.12) \times \sin(\alpha)) / \cos(\text{latitude}_{\text{average}})$
   - Update both coordinate variables:
     - `redLatLng = new LatLng(redLatLng.getLatitude() + deltaLat1m, redLatLng.getLongitude() + deltaLon1m);`
     - `greenLatLng = new LatLng(greenLatLng.getLatitude() + deltaLat1m, greenLatLng.getLongitude() + deltaLon1m);`
   - Since both points are translated by the exact same 2D vector, the relative distance between them is preserved perfectly at **1.0 km**.

4. **UI Markers & Camera Updating**:
   - Update markers on the map:
     - `redSymbol.setLatLng(redLatLng);`
     - `greenSymbol.setLatLng(greenLatLng);`
     - `symbolManager.update(redSymbol);`
     - `symbolManager.update(greenSymbol);`
   - Center the camera on the midpoint of the moving points for a smooth, premium tracking effect:
     - `LatLng midpoint = new LatLng((redLatLng.getLatitude() + greenLatLng.getLatitude()) / 2.0, (redLatLng.getLongitude() + greenLatLng.getLongitude()) / 2.0);`
     - `mapLibreMap.animateCamera(CameraUpdateFactory.newLatLng(midpoint));`

5. **Lifecycle Management**:
   - Start the loop once the map style and initial symbols are fully ready.
   - Properly stop/pause the loop in `onPause()` / `onDestroy()` by removing handler callbacks to prevent any background resource or memory leaks.
