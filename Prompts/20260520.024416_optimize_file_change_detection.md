# Implementation Plan - Optimize File Change Detection via Timestamps

This plan details the steps to enhance our dynamic GPS tracking polling loop in `MainActivity.java` to explicitly track when files have changed using their filesystem modification timestamps.

## 1. Objectives & Requirements
- **Efficient Tracking**: Maintain a memory of the last modified timestamp for each session file.
- **Explicit Change Check**: Every second, check if a file is new or if its `lastModified()` timestamp has increased.
- **Dynamic Updates**: Only read, parse, and recreate/update markers when a change is detected, reducing I/O and CPU overhead.
- **Graceful Deletion Handling**: If a file is deleted from the directory, remove its corresponding map marker.

## 2. Technical Details
- We will add a `private final java.util.Map<String, Long> fileLastModifiedMap = new java.util.HashMap<>();` field to `MainActivity.java`.
- In the polling loop:
  - Scan all files in `/sdcard/Vypeensoft/Friends_Location_Tracker/sessions/`.
  - For each file, compare its current `lastModified()` timestamp with `fileLastModifiedMap.get(fileName)`.
  - If it's new or the timestamp is greater:
    - Read and parse the file.
    - Add/update the corresponding `UserLocation` and update the map marker.
    - Update `fileLastModifiedMap` with the new timestamp.
  - If any tracked file is no longer in the folder:
    - Remove the marker.
    - Clean up its state.
